  • configureSearchIndex(hotMeshClient, search?): Promise<void>

  • For those deployments with a redis stack backend (with the FT module), this method will configure the search index for the workflow. For all others, this method will exit/fail gracefully and not index the fields in the HASH. All values are searchable via HKEYS/HSC/HGET

    Parameters

    • hotMeshClient: HotMesh

      the hotmesh client

    • Optionalsearch: WorkflowSearchOptions

      the search options

    Returns Promise<void>

     index: 'my_search_index',
     prefix: ['my_workflow_prefix'],
     schema: {
      field1: { type: 'TEXT', sortable: true },
      field2: { type: 'NUMERIC', sortable: true }
     }
    }
